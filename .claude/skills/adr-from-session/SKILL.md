## 役割

このセッションで合意された設計判断を抽出し、`docs/adr/NNNN-<kebab-slug>.md` として新規ファイルを追加し、`docs/ADR.md` インデックスを更新する。

**重要:** `docs/ADR.md` はインデックスファイル。詳細は `docs/adr/` 配下の個別ファイルに書く。既存の `docs/spike-*.md`（DB / ORM / IaC 選定）は確定済みの調査記録であり、関連する ADR からはリンクで参照する。

## 手順

### Step 1: 既存インデックスとファイル一覧を読む

`docs/ADR.md` を Read で全文読み、最終 ADR 番号・既存カテゴリを把握する。`docs/adr/` 配下のファイル名から最大番号を確認する。番号は `ADR-NN`（表示は2桁、ファイル名は `NNNN` 4桁ゼロパディング）。

### Step 2: 抽出対象を判別

**抽出対象（暗黙合意も含む）:**

- 「採用」「決定」「やめる」など**明示的に合意**された方針
- 複数案が提示された末に Claude が**実装に踏み切った**選択（暗黙合意とみなす）
- CLAUDE.md / 既存スキル / docs/*.md と**異なる判断**をした箇所
- 既存 ADR と矛盾する/上書きする判断
- 将来の読み手が「なぜ?」と思いそうな箇所
- ハーネス・ツール・運用フローに関する判断（コード以外も対象）

**抽出しない:**

- 既存 ADR・spike-*.md にすでに記録済みの判断
- 純粋な調査・質問応答・コード説明のみのセッション
- バグ修正・タイポ修正など方針変更を伴わない作業
- 「とりあえず」「あとで決める」など暫定保留の議論

### Step 3: 個別 ADR ファイルを作成

判断が一つ以上見つかった場合のみ。判断1件につき1ファイル。

**ファイル名:** `docs/adr/NNNN-<kebab-slug>.md`（`NNNN` = 既存最大+1、`<kebab-slug>` = 英語 kebab-case）

**ファイル形式（重要な判断: Michael Nygard 形式）:**

```markdown
---
id: ADR-NN
title: 判断テーマ
category: カテゴリ名
status: 暫定
date: YYYY-MM-DD
---

# ADR-NN: 判断テーマ — 採用した選択

**ステータス:** 暫定

## コンテキスト

（何が起きていてなぜ判断が必要になったか。暗黙合意の場合は「ユーザー明示の合意ではなく〇〇という対話の流れから決定とみなした」と明記）

## 検討した選択肢

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| 採用案 | ... | ... |
| 却下案 | ... | ... |

## 決定

採用した方針を一文＋具体。

## 結果

- 良い帰結
- 注意すべき帰結
- 関連: [#PR番号] / [docs/path] / [spike-*.md]

## 関連

- [ADR-MM](./NNNN-other.md) — 関連する判断
```

**軽微な判断（簡略版）:** `## コンテキスト`〜`## 結果` を省略し、`## 決定` と `## 根拠` のみで可。

### Step 4: docs/ADR.md インデックスを更新

**4-a) 「一覧」テーブルに1行追加（ID 昇順を維持）:**

```markdown
| [ADR-NN](./adr/NNNN-slug.md) | カテゴリ名 | 判断テーマ | 採用した選択 | 暫定 |
```

**4-b) 「カテゴリ別」セクションの該当カテゴリに1行追加。** カテゴリが新規なら `### 新カテゴリ名` を追加。

**4-c) 関連 ADR があれば双方向リンクを張る**（新規ファイルと既存ファイル両方の `## 関連` を Edit）。

### Step 5: 一切判断が抽出できなければ

ファイルを変更せず、「該当する判断なし。ADR は追加しませんでした。」とだけ返す。

- データ・DB
- インフラ・運用

- テスト
- ハーネス・ツール

## 抽出スタイルの注意

- **暗黙合意の抽出根拠を明示する**（後から人間がレビューしやすく）
- **ステータスは原則「暫定」**。確定後にユーザーが手動で frontmatter の `status:` を `採用` に書き換える運用
- **slug は英語 kebab-case**（ASCII 英小文字とハイフンのみ）
- **日付は frontmatter の `date:`** で記録（本文に重複させない）
- 既存 `docs/spike-*.md` と重複する判断は ADR 化せず、必要なら ADR から spike へリンク
