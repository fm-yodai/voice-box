# voice-box プロジェクト知識

匿名意見箱プラットフォーム。従業員が公開画面に匿名でフィードバック・バグ報告・提案を投稿でき、
システム内部では悪用防止のため投稿者IDを追跡する。投稿と対応ステータスを公開画面で可視化する
ことで（透明性）、投稿が放置されないことを担保する。

## 思想の核（必ず踏まえる）

- **根本課題は組織変革**: 創業期の「個人事業主の集まり」意識が残り、"組織的にものごとを行う"
  思考が欠如している。voice-box はその解決策の**一手段**（声→改善ループで組織的意識を醸成）
- **姉妹施策**: ありがとうカード（心理的安全性・MVV浸透）／業務課題チャレンジプログラム
  （挑戦推奨・個人改善の組織還元）と同じ根本課題への別レバー。連携して効く
- **最大の差別化は透明性**（公開画面で投稿と対応状況が見える）
- **AI は半自動支援はする・全自動はしない**: 建設的議論のための介在は Goal、全自動モデレーション
  ／全自動運用は Non-Goal
- 詳細は [references/inception-deck.md](references/inception-deck.md)（10 問の全体像・やらないこと
  リスト・リスク・トレードオフ。未確定項目は `[要確認]`）

## 最重要原則

- **匿名表示ファースト**: 公開画面・API レスポンスから投稿者情報（userId/email/displayName）を必ず除外する
- **内部トレーサビリティ**: 内部DBには userId・投稿日時・IPアドレスを保持（脅威・ハラスメント・法令違反時の追跡用）
- **二重承認**: 投稿者追跡は厳格な承認プロセス（dual authorization）を経てのみ実施
- **監査ログ**: 追跡操作は監査ログに記録し 2 年間保管

## ロール（GitHub ラベルとも対応: role:member / role:moderator / role:admin）

| ロール | 権限 |
|--------|------|
| member | 投稿の作成・閲覧（匿名） |
| moderator | 投稿のステータス変更・返信・モデレーション |
| admin | 追跡承認・ユーザー管理・監査ログ閲覧 |

## 実装時の判断指針

- 投稿者情報を公開経路に流す変更は **匿名性違反**。レビューで必ず指摘する
- 追跡系機能は governance.md の承認プロセスに準拠しているか確認する
- 仕様が governance.md と矛盾する場合、コードではなくドキュメントを正とし、矛盾を必ず表面化する
