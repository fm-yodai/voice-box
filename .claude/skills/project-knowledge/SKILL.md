# voice-box プロジェクト知識

匿名意見箱プラットフォーム。従業員が公開画面に匿名でフィードバック・バグ報告・提案を投稿でき、
システム内部では悪用防止のため投稿者IDを追跡する。SLA 駆動の応答システムで投稿が放置されない
ことを保証する。

## 最重要原則

- **匿名表示ファースト**: 公開画面・API レスポンスから投稿者情報（userId/email/displayName）を必ず除外する
- **内部トレーサビリティ**: 内部DBには userId・投稿日時・IPアドレスを保持（脅威・ハラスメント・法令違反時の追跡用）
- **二重承認**: 投稿者追跡は厳格な承認プロセス（dual authorization）を経てのみ実施
- **監査ログ**: 追跡操作は監査ログに記録し 2 年間保管

## ロール（GitHub ラベルとも対応: role:member / role:moderator / role:admin）

| ロール | 権限 |
|--------|------|
| member | 投稿の作成・閲覧（匿名） |
| moderator | 投稿のステータス変更・返信・モデレーション |
| admin | 追跡承認・ユーザー管理・監査ログ閲覧 |

## 実装時の判断指針

- 投稿者情報を公開経路に流す変更は **匿名性違反**。レビューで必ず指摘する
- 追跡系機能は governance.md の承認プロセスに準拠しているか確認する
- 仕様が governance.md と矛盾する場合、コードではなくドキュメントを正とし、矛盾を必ず表面化する
