# ガバナンスポリシー

## 匿名表示と投稿者追跡に関するポリシー

### 1. 基本方針

#### 1.1 匿名表示の原則
- voice-boxでは、すべてのユーザーの投稿は**原則として匿名で表示**されます
- ユーザーが安心して率直な意見を表明できる環境を提供することを最優先とします
- 公開される投稿には、投稿者を特定できる情報（ユーザー名、メールアドレス等）は表示されません

#### 1.2 投稿者情報の内部保持
- システムの健全性と責任の所在を明確にするため、**システム内部では投稿者のIDを保持**します
- 保持される情報:
  - ユーザーID（内部識別子）
  - 投稿日時
  - 投稿内容
  - IPアドレス（セキュリティ目的）
- これらの情報は、通常の運用では公開されず、厳重に管理されます

### 2. 投稿者追跡が可能な条件

以下のいずれかに該当する場合、投稿者の追跡・特定を行うことができます:

#### 2.1 追跡可能なケース
1. **脅迫・恐喝行為**
   - 他者への脅迫、恐喝、危害を加える予告など

2. **個人攻撃・ハラスメント**
   - 特定個人への誹謗中傷、プライバシー侵害
   - 執拗な嫌がらせ、ストーキング行為

3. **法令違反**
   - 日本国の法令に違反する行為
   - 著作権侵害、名誉毀損、その他の違法行為

4. **システム不正利用**
   - 自動化ツールによるスパム投稿
   - システムの脆弱性を悪用した攻撃
   - なりすまし、アカウント乗っ取り

5. **公共の安全に関わる緊急事態**
   - 自傷・他害の具体的な予告
   - 公共の安全を脅かす情報

### 3. 追跡の承認プロセス

#### 3.1 承認者
投稿者追跡の最終承認は、以下の役職者が行います:

**第一承認者:**
- プロジェクトオーナー（Project Owner）

**第二承認者（以下のいずれか）:**
- システム管理責任者（System Administrator）
- コミュニティマネージャー（Community Manager）

**緊急時:**
- 人命に関わる緊急事態の場合、第一承認者の単独判断で追跡を実施可能
- ただし、事後24時間以内に第二承認者への報告が必須

#### 3.2 追跡手順

1. **報告受付**
   - 問題投稿の報告を受付（運営メール、専用フォーム等）
   - 報告内容の記録（報告者、報告日時、問題投稿のURL/ID）

2. **初期調査**
   - 問題投稿の内容確認
   - 追跡条件（2.1項）への該当性を判断
   - 証拠の収集（スクリーンショット、ログ等）

3. **承認申請**
   - 追跡申請書の作成
     - 対象投稿のID
     - 追跡理由（該当条項）
     - 緊急度の評価
   - 第一承認者・第二承認者への申請

4. **承認判断**
   - 承認者による審査（原則48時間以内）
   - 承認/却下の決定
   - 決定理由の記録

5. **追跡実施**
   - 承認後、最小限の範囲で投稿者情報を開示
   - 必要に応じて関係者（法執行機関等）への情報提供

6. **事後処理**
   - 追跡結果の記録
   - 必要に応じた措置（アカウント停止、投稿削除等）
   - 報告者への結果通知（個人情報に配慮）

### 4. 監査ログ

#### 4.1 記録対象
すべての投稿者追跡操作は、監査ログに記録されます:

**記録される情報:**
- 操作日時
- 操作者（追跡を実施したアカウント）
- 対象投稿ID
- 追跡理由
- 承認者
- 開示された情報の範囲
- 追跡結果（概要）

#### 4.2 ログの保管
- 監査ログは**5年間保管**されます
- アクセス権限は、プロジェクトオーナーとシステム管理責任者に限定
- 年次レビューで適切性を確認

#### 4.3 透明性レポート
- 半年ごとに追跡実績の統計を公開（匿名化）
  - 追跡件数
  - 追跡理由の分類
  - 措置の種類
- 個別ケースの詳細は公開しない

### 5. 利用者への周知

- 本ポリシーは、サービス利用規約の一部として公開されます
- 初回利用時に、本ポリシーへの同意を求めます
- ポリシー変更時は、事前に告知し、ユーザーの意見を募ります

### 6. ポリシーの見直し

- 本ポリシーは、年1回定期的に見直しを行います
- 重大なインシデント発生時は、臨時での見直しを実施
- 見直し結果は、運営メンバー全員の合意を経て更新されます
